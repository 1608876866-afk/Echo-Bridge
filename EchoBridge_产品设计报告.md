## 一、产品概述

### 1.1 产品定位

共鸣之桥是一款面向轻中度社交焦虑人群的数字化干预工具。产品严格遵循认知行为疗法 (Cognitive Behavioral Therapy, CBT) 的"闭环干预"原则，通过认知重构、暴露疗法和注意力偏差修正三大核心机制，帮助用户在低压、安全的数字环境中逐步重建社交信心。

### 1.2 目标用户

- **年龄范围**：18-35 岁
- **症状程度**：轻中度社交焦虑（非病理性社交恐惧症早期）
- **典型场景**：职场会议压力、聚会后反刍、日常社交回避
- **排除标准**：精神分裂症、重度抑郁发作期、有自伤/自杀高风险

### 1.3 核心价值

- **零门槛**：纯浏览器运行，无需安装，无外部依赖
- **隐私优先**：所有数据仅存储在用户本地浏览器，不上传任何服务器
- **个性化**：基于 LSAS 测评结果 + AI 动态生成定制化干预方案
- **人机陪伴**：AI 驱动的虚拟宠物伙伴，融合 CBT 引导与游戏化激励

## 二、医学逻辑与治疗框架

### 2.1 核心作用机制 (Mechanism of Action)

本产品严格遵循循证心理治疗的三项核心机制：

#### 2.1.1 认知重构 (Cognitive Restructuring)

针对社交焦虑中的**负面自动思维**（Automatic Negative Thoughts, ANTs），通过苏格拉底式提问和认知偏差识别，帮助用户区分"事实"与"想象"。

**常见认知偏差识别清单**：
- 灾难化思维 (Catastrophizing)：将小事件预测为最坏结果
- 读心术 (Mind Reading)：假设知道他人对自己的负面评价
- 非黑即白 (All-or-Nothing Thinking)：将社交结果二分为完美/失败
- 过度概括 (Overgeneralization)：将单次负面经历推广为永久模式
- 情绪推理 (Emotional Reasoning)：因为感到紧张所以认为事情很糟糕
- 个人化归因 (Personalization)：将他人的行为过度归咎于自己

#### 2.1.2 暴露疗法 (Exposure Therapy)

设计**分级社交暴露任务**（Hierarchy of Social Exposure），从低风险的虚拟/观察行为逐步过渡到中等风险的轻度互动：

| 等级 | 风险程度 | 示例任务 |
|------|---------|---------|
| Lv1 | 极低 | 在电梯里向邻居点头微笑 |
| Lv1 | 极低 | 对便利店收银员说"谢谢" |
| Lv1 | 极低 | 给朋友发一条文字消息 |
| Lv2 | 中等 | 在会议中说一句话 |
| Lv2 | 中等 | 在咖啡馆点餐时主动询问推荐 |
| Lv2 | 中等 | 在社交媒体上发表评论 |
| Lv3 | 较高 | 主动发起 2 分钟以上对话 |
| Lv3 | 较高 | 在小组讨论中表达不同意见 |
| Lv3 | 较高 | 拒绝不合理的请求 |

#### 2.1.3 注意力偏差修正 (Attentional Bias Modification, ABM)

训练用户将注意力从**自我审视**（内部聚焦）转向**外部环境**（外部聚焦），减少社交场合中的自我监控行为。典型训练包括观察对方衣着、注意语调变化、计数环境声音等。

### 2.2 治疗目标

- **主要终点**：降低 Liebowitz 社交焦虑量表 (LSAS) 评分
- **次要终点**：减少社交回避行为频率，提高社交后"效能感自评"
- **过程指标**：训练任务完成率、日记记录频率、AI 对话参与度

### 2.3 风险边界与安全保障

| 机制 | 措施 |
|------|------|
| 危机监测 | 所有输入框实时扫描自伤/自残关键词，触发温和危机干预模式 |
| 禁忌症提示 | 欢迎页明确标注不适用于精神分裂症或重度抑郁发作期 |
| 转介机制 | LSAS > 80 分显示专业转介建议 |
| 隐私保护 | 知情同意书 + LocalStorage + 数据导出/删除权 |
| 非医疗声明 | 明确声明产品为辅助训练工具，不替代专业诊断 |

### 2.4 危机干预流程

当系统检测到敏感内容时：
1. 页面以 2 秒渐变过渡为柔和暖色调全屏模式（非突然弹窗）
2. 显示共情性文字与全国心理援助热线
3. 提供"和 AI 伙伴聊聊"和"我知道了"两个选项
4. 用户可自主退出，不强制锁定

## 三、功能模块设计

### 3.1 整体架构

```
[欢迎页] → [知情同意] → [LSAS首次测评] → [结果分析 + AI生成方案]
                                                    ↓
                              ┌─── 底部导航栏 ──────────────────┐
                              │ 训练 │ AI伙伴 │ 社区 │ 日记    │
                              └────────────────────────────────┘
                                                    │
                                      右上角头像 → [个人中心]
                                        ├── 数据报告
                                        ├── 急救工具箱
                                        └── 设置
```

### 3.2 模块一：欢迎页与知情同意

**功能要点**：
- 品牌展示（渐变鼠尾草绿背景，"共鸣之桥 EchoBridge" + 副标题"重建社交连接的温柔练习"）
- 可滚动阅读的知情同意书，涵盖六大隐私与安全声明
- 强制勾选同意后方可进入，确保合规

**知情同意书核心内容**：
1. 数据本地存储声明（不上传服务器）
2. LocalStorage 风险提示（清除缓存导致数据丢失，建议定期导出）
3. AI 服务数据传输告知（API 调用说明）
4. 适用与不适用人群界定
5. 非医疗建议声明
6. 用户权利（导出/删除/停止使用）

### 3.3 模块二：LSAS 测评系统

**量表规格**：
- **量表名称**：Liebowitz Social Anxiety Scale (LSAS)
- **题数**：完整版 24 题（11 个社交互动场景 + 13 个表现场景）
- **评分维度**：每题评分两个维度
  - 恐惧/焦虑程度：0（无）→ 1（轻度）→ 2（中度）→ 3（重度）
  - 回避频率：0（从不）→ 1（偶尔）→ 2（经常）→ 3（总是）
- **总分范围**：0-144
- **填写时长**：约 5-8 分钟

**评分等级与处理策略**：

| 分数区间 | 等级 | 系统响应 |
|---------|------|---------|
| 0-54 | 轻度社交焦虑 | 生成标准 7 天训练计划 |
| 55-65 | 中度社交焦虑 | 生成强化训练计划 + 建议定期复评 |
| 66-80 | 较重社交焦虑 | 生成强化计划 + 提示关注进展 |
| >80 | 建议转介 | 显示专业支持建议 + 一键呼叫热线 |

**复评提醒机制**：
- 首次测评后设置复评频率（每周/每月）
- 用户自主选择提醒时段（早晨 8:00 或睡前 21:00）
- 温和半透明提示条，非强制弹窗，用户可选择延迟

### 3.4 模块三：定制化干预方案生成

**AI 驱动方案生成流程**：
1. 输入：LSAS 各维度得分 + 用户自选目标场景（职场/学校/日常社交/亲密关系/公共场合）
2. 调用 DeepSeek API 生成 7 天结构化训练计划
3. 输出：每天 2-3 个任务，覆盖三种干预机制

**用户可调节参数**：
- 每日任务数量：1-4 个（滑动条调节）
- 难度偏移：-2（最简单）至 +2（最困难）
- 单任务替换：点击可换为预设库或 AI 生成备选任务

**预设任务库（离线备用）**：
- 认知重构类：7 个标准化任务
- 暴露任务类：12 个分级任务（Lv1-Lv3）
- 注意力训练类：6 个训练任务

### 3.5 模块四：日常打卡训练 (Day Plan)

**主界面构成**：
- 顶部：每日"社交电量"自评滑块（0-10，带情绪表情映射）
- 中部：当日任务卡片列表（类型标签 + 难度等级 + 描述 + 完成状态）
- 连续打卡天数计数器

**任务执行闭环**：
1. 用户阅读任务说明
2. 在现实中执行任务（暴露/注意力任务需线下完成）
3. 返回应用完成打卡
4. 书写感想（输入框提示："任务完成得怎么样？有什么感受？有没有出现让你在意的想法？"）
5. AI 分析感想 → 生成鼓励 + 识别认知偏差 + 认知重构引导
6. 公开/私密切换（选择是否分享到社区）

### 3.6 模块五：AI 陪伴对话 (AI Companion)

**AI 角色设计**：
- 虚拟宠物形态（CSS 绘制，支持柴犬/猫咪/小鹿三种形态切换）
- 用户可自定义名字（默认"桥桥"）
- 宠物情绪随用户活跃度变化（活跃→精神饱满，3 天未打卡→显示期待表情）

**预训练 System Prompt**：
AI 伙伴经过专业 CBT 治疗框架训练，核心能力包括：
- CBT 认知行为疗法框架（贝克模型）
- 苏格拉底式提问技巧
- 社交焦虑常见认知偏差识别（6 大偏差类型）
- 温暖、接纳、不评判的语调风格
- 安全边界：识别自伤倾向 → 共情 + 温和转介引导

**对话界面**：
- 聊天气泡式布局
- 底部快捷情景卡片（可滑动）：
  - "我马上要参加一个会议，很紧张"（场景 A：预热）
  - "我刚才说错话了，一直在想"（场景 B：复盘）
  - "我想练习社交，但不知道从哪里开始"（场景 C：练习）
  - "我最近不太想出门见人"（情绪支持）

**四种场景模式 AI 行为**：
- **预热模式**：认知准备清单 + 简短呼吸练习 + 鼓励语句
- **复盘模式**：引导分析"事实 vs 想象" + 自我关怀引导 + 学习点提炼
- **练习模式**：根据用户描述生成 2-3 个低门槛练习建议
- **情绪支持模式**：共情倾听 + 温和行为激活建议

### 3.7 模块六：日记模块 (Daily Journal)

**结构化引导模板**：
输入框内预置半透明引导文字，采用 CBT 标准记录格式：

> 场景：今天在什么地方，和谁在一起？
> 情绪：当时你的感受是什么？（紧张/尴尬/害怕/其他）
> 躯体反应：身体有什么感觉？（心跳加快/脸红/出汗/其他）
> 行为表现：你做了什么？说了什么？
> 自动思维：那一刻你脑海中闪过了什么念头？

**AI 分析功能**：
- 用户完成日记后点击"AI 分析"
- 系统调用 DeepSeek API 分析内容
- 输出：三个挑战性视角 + 识别到的认知偏差 + 鼓励反馈
- 分析结果保存到日记条目中

**历史回顾**：
- 按日期排列的日记列表
- 点击展开查看完整内容与 AI 分析
- 每条日记支持公开/私密设置

### 3.8 模块七：社区模块 (Community)

**设计说明**：
基于纯前端架构的技术限制，当前社区为**本地模拟社区**。公开的打卡和日记存储在用户本地 LocalStorage 中，在社区页面以信息流形式展示。该设计为未来接入真正后端服务的架构预留。

**界面构成**：
- 信息流形式（类朋友圈/小红书布局）
- 每张卡片：用户昵称 + 头像 + 内容类型标签 + 正文 + 时间
- 点赞功能（点击切换 + 计数显示）
- 评论功能（展开评论区 + 添加评论）
- 排序：最新在前

**与打卡/日记的联动**：
- 打卡完成页和日记编辑页均有隐私开关
- 选择"公开"→ 内容自动发布到社区流

### 3.9 模块八：数据报告 (Recovery Log)

**顶部可视化面板**：

1. **焦虑趋势折线图**（Canvas 绘制）

**每日详情面板**：

## 四、技术架构

### 4.1 技术方案总览

| 维度 | 方案 |
|------|------|
| 技术形态 | 单文件 Web 应用 (Single Page Application) |
| 前端技术 | 原生 HTML5 + CSS3 + Vanilla JavaScript |
| 外部依赖 | 零依赖，无任何第三方库/框架 |
| API 服务 | DeepSeek API (chat/completions) |
| 数据持久化 | 浏览器 LocalStorage |
| 图表渲染 | Canvas API (原生) |
| 运行方式 | 本地 HTTP 服务器 (推荐) 或直接文件打开 |

### 4.2 文件结构

```
echobridge.html (单文件，~3500 行)
├── <style>  CSS 设计系统 (~900 行)
│   ├── CSS 变量系统 (颜色、圆角、阴影、字体、过渡)
│   ├── 组件样式 (卡片、按钮、导航、弹窗、输入框)
│   ├── 模块样式 (训练、对话、社区、日记、报告、急救)
│   └── 响应式断点 (768px / 1024px)
├── <body>   HTML 结构 (~500 行)
│   ├── 应用外壳 (Header + Main Content + Bottom Nav)
│   ├── 11 个视图容器 (display:none 切换)
│   ├── 全屏呼吸动画层
│   ├── 危机干预覆盖层
│   ├── 抽奖弹窗
│   └── Toast / 确认弹窗
└── <script> JavaScript (~2100 行)
    ├── 数据层 (DB 模块：LocalStorage CRUD)
    ├── 安全层 (Safety 模块：敏感词实时检测)
    ├── API 层 (DeepSeek API 调用封装)
    ├── 路由系统 (SPA 视图管理)
    ├── LSAS 测评引擎
    ├── 预设任务库
    ├── UI 渲染函数 (11 个模块)
    ├── Canvas 图表引擎
    ├── 呼吸动画引擎 (requestAnimationFrame)
    ├── 激励系统 (打卡/里程碑/抽奖)
    └── 初始化入口
```

### 4.3 设计系统

**色彩规范**（心理学依据）：

| 颜色角色 | 色值 | 心理学含义 |
|---------|------|-----------|
| 主色（鼠尾草绿） | #A3C1AD | 安全感、平静、自然 |
| 主色浅 | #C5D9C9 | 柔和、舒适 |
| 主色深 | #7FA089 | 稳定、可靠 |
| 背景（旧蕾丝白） | #FDF5E6 | 温润不刺眼、怀旧安全感 |
| 背景渐变 | #E8F0E8 → #FFFFFF | 低压、空气感 |
| 警示色（淡珊瑚） | #E9967A | 温和提醒，不制造恐慌 |
| 文字色 | #3C3C3C | 柔和深灰，非纯黑 |
| 卡片色 | #FFFFFF | 清晰区隔 |

**字体系统**：
- 主字体：`PingFang SC, -apple-system, BlinkMacSystemFont, Segoe UI, Noto Sans SC, sans-serif`
- 字间距：`0.5px`（略加宽提升易读性）

### 4.6 AI API 调用设计

**服务商**：DeepSeek API

**调用点与频率**：

| 调用场景 | 每次消耗 | 触发方式 |
|---------|---------|---------|
| 生成 7 天训练计划 | ~2000 tokens | 测评完成后手动触发 |
| 打卡感想反馈 | ~500 tokens | 打卡写感想提交时 |
| 日记 AI 分析 | ~600 tokens | 写日记后手动触发 |
| AI 陪伴对话 | ~600 tokens/次 | 自由对话，按需触发 |

**成本控制**：
- 提供 AI 增强模式开关，关闭后使用预设模板库
- 所有调用均为用户手动触发，无自动后台调用
- 单日预估最大消耗：约 4000-8000 tokens

**System Prompt 设计**（AI 预训练核心）：
- 角色定位：CBT 训练有素的社交焦虑康复陪伴伙伴
- 语调风格：温暖、接纳、不评判
- 核心能力：认知偏差识别、苏格拉底式提问、行为激活建议
- 安全边界：自伤倾向 → 共情 → 温和转介引导
- 回复限制：简洁精炼，每次 150-800 字

### 4.7 安全性设计

**关键词监测系统**：
- 监测位置：所有文本输入框（日记、对话、感想）
- 监测方式：正则实时匹配（input 事件监听）
- 关键词库：自杀、自伤、不想活、结束生命、伤害自己、割腕、跳楼、安眠药过量等 15+ 关键词
- 触发行为：2 秒渐变全屏危机干预模式（不弹窗/不锁定）

**隐私保护措施**：
- 全量数据 LocalStorage 本地存储
- 知情同意书强制确认
- 数据导出/删除权
- API Key 由用户自行提供和保管

## 五、视觉设计规范

### 5.1 设计理念

**核心关键词**：低压、静谧、安全、温润

**设计原则**：
- 避免强烈的视觉对比和刺激性色彩
- 所有过渡动画使用缓动曲线，避免突兀变化
- 大面积留白，降低信息密度
- 卡片作为主要信息容器，提供清晰的空间区隔
- 圆角作为统一设计语言（24px 卡片 + 16px 按钮 + 50px 胶囊按钮）

### 5.2 组件规范

| 组件 | 规格 |
|------|------|
| 卡片 | border-radius: 24px, 阴影: 0 2px 12px rgba(0,0,0,0.06) |
| 主按钮 | 胶囊形, 主色背景, 白色文字, 悬停上浮 1px |
| 次要按钮 | 胶囊形, 白色背景, 主色边框, 悬停填充 |
| 导航栏 | 毛玻璃效果 (backdrop-filter), 64px 高度 |
| 输入框 | 圆角 8-12px, 1.5px 边框, 聚焦时变为主色 |
| 弹窗 | 底部滑出式 Sheet, 半透明背景遮罩 |

### 5.3 动效规范

- 默认过渡：`0.3s cubic-bezier(0.4, 0, 0.2, 1)`
- 卡片交互：点击缩放至 0.98x
- 按钮悬停：上浮 1px + 阴影加深
- 页面切换：无过渡（直接显隐），避免眩晕
- 呼吸动画：`requestAnimationFrame` 驱动，精确帧控制
- 提供"减少动效"全局开关（设置 `animation-duration: 0.01ms`）

## 七、局限性与未来规划

### 7.1 当前版本局限性

1. **社区为模拟实现**：受限于纯前端架构，当前社区基于 LocalStorage 模拟，无法实现真正的多用户互动。未来需引入后端服务实现真实社区。

2. **AI 依赖外部服务**：AI 功能依赖 DeepSeek API，需要用户自行提供 API Key 并承担调用费用。离线模式仅能使用预设模板库。

3. **LocalStorage 持久化风险**：浏览器清除缓存、无痕模式等会导致数据丢失。已通过知情同意告知和数据导出功能缓解。

4. **单设备限制**：数据不跨设备同步，用户更换设备需要手动导入导出。

5. **缺乏临床验证**：当前为原型版本，尚未经过随机对照试验 (RCT) 的临床验证。

### 7.2 未来迭代方向

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | 后端服务 + 真实社区 | Node.js/Python 后端，数据库持久化，真多用户社区 |
| P0 | 数据云同步 | 用户账号系统 + 加密云同步 |
| P1 | 临床验证 | 设计 RCT 试验方案，收集疗效数据 |
| P1 | 语音对话 | 接入语音识别 + TTS，支持语音版 AI 对话 |
| P2 | 多端适配 | PWA 封装，支持离线使用和桌面快捷方式 |
| P2 | 更多量表 | 整合 GAD-7、PHQ-9 等配套评估工具 |
| P3 | 生物反馈集成 | 对接心率传感器，实现生理数据驱动的干预 |
